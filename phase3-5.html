GenAI Connect: Configuration & Walkthrough

~2h

Step-by-step configuration of Pega GenAI Connect in App Studio. Set up model connections, configure routing, test prompts, and deploy to production channels.

Configuration Steps
  • Navigate to App Studio → Settings → GenAI Connect
  • Add LLM provider (Azure OpenAI / Google Vertex / Anthropic)
  • Enter API endpoint, key/secret, and deployment name
  • Configure default model and fallback chain
  • Set token budget and cost alert thresholds
  • Enable Prompt Masking for PII fields
  • Test connection with built-in prompt sandbox
  • Publish configuration to environments

Topic Guide

Connecting Models to Real Workflow Actions

GenAI Connect is important because model calls only become useful in enterprise delivery when they are embedded into real application flows. A walkthrough of the configuration is not just about where to click. It teaches how prompts, data mappings, provider settings, and downstream actions come together into a repeatable pattern. Builders need to understand what data enters the prompt, how the model response is parsed, and what part of the workflow consumes the result.

Configuration discipline matters here because weak mappings and unclear prompt contracts create fragile solutions. A connector should behave like any other production integration: explicit inputs, explicit outputs, clear exception handling, and auditability. In Pega, the point is to move from isolated experimentation into reliable workflow components that business teams can reuse, test, and support.

  • Prompt templates should be parameterized and versioned instead of hard-coded inline.
  • Data mappings must make the source and target of each field unambiguous.
  • Error handling should define what happens if the provider fails or returns unusable output.
  • Workflow integration is what turns the model call into business value.
Worked Example

GenAI Connect Example: Case Summary Drafting

Suppose a service case reaches escalation and the operations manager needs a concise summary before taking over. GenAI Connect can map the case history, customer sentiment, open tasks, and recent correspondence into a structured prompt. The model generates a first-draft summary, and the case workflow stores that output in a review field for an agent to approve or edit.

1

Map Inputs

Case data, notes, and SLA status are bound into the prompt template.

2

Call Model

GenAI Connect sends the request to the configured provider.

3

Store Output

The draft summary returns to a case field instead of being published immediately.

4

Approve

A human user reviews the draft before sending or saving it.

Design lesson: the safest early production patterns use GenAI Connect to draft or recommend inside a workflow, then keep the final decision with a human or a deterministic business rule.
