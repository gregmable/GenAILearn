Glossary

Key terms and abbreviations used throughout the GenAI Learning Path. Click any card to read its definition.

Generative AI GenAI
A class of AI models that can generate new content — text, images, code, audio — by learning the statistical patterns of training data. Examples include GPT-4 and Gemini.
Analytical AI
Traditional ML models that classify, predict, or score existing data rather than generating new content. Pega's Next-Best-Action and propensity models are Analytical AI.
Large Language Model LLM
A deep neural network trained on massive text corpora to understand and generate human language. LLMs power most modern GenAI products.
Token
The basic unit of text an LLM processes — roughly one word or ~4 characters. Both prompt and response text consume tokens, and model limits are expressed in token counts.
Context Window
The maximum number of tokens an LLM can "see" at once — both input and output combined. Content outside this window is invisible to the model during a given call.
Fine-tuning
Continued training of a pre-trained LLM on a curated, domain-specific dataset to shift its behavior — improving accuracy on narrow tasks at a lower inference cost than prompt engineering.
Retrieval-Augmented Generation RAG
A pattern where relevant documents are retrieved from a knowledge store and injected into the LLM prompt at query time, grounding the response in authoritative content without retraining.
Vector Embedding
A dense numeric representation of a piece of text (or image) in high-dimensional space, where semantically similar items cluster together. The foundation of semantic search and RAG retrieval.
Prompt Engineering
The practice of designing and iterating on the text input to an LLM — including system instructions, few-shot examples, and chain-of-thought cues — to achieve reliable, high-quality outputs.
Hallucination
When an LLM generates confident-sounding but factually incorrect or fabricated information. Mitigated through RAG grounding, temperature reduction, and output validation.
Temperature
A sampling parameter (0–2) that controls output randomness. Lower values produce more deterministic, focused responses; higher values increase creativity and variability.
Reinforcement Learning from Human Feedback RLHF
A training technique that uses human preference rankings to fine-tune LLM behavior, steering models toward helpful, harmless, and honest responses.
Model Context Protocol MCP
Anthropic's open standard for connecting AI agents to external tools, data sources, and APIs via a uniform interface — enabling agents to act across enterprise systems.
Agent-to-Agent Protocol A2A
Google's open protocol enabling independent AI agents built on different frameworks to communicate, delegate tasks, and collaborate in multi-agent workflows.
Prompt Injection
An adversarial attack where malicious text in the environment (e.g., a web page or document) overrides the original system instructions and hijacks the agent's behavior.
Human-in-the-Loop HITL
A governance pattern requiring human review or approval before an AI agent executes a consequential action, providing an override checkpoint for high-risk steps.
Chunking
The process of splitting large documents into smaller text segments before embedding, balancing retrieval granularity against context completeness in a RAG pipeline.
Semantic Cache
A cache layer that stores LLM responses keyed by the semantic meaning of the query (not exact text). Similar future queries hit the cache, reducing latency and API cost.
Faithfulness / Groundedness
RAG quality metrics: faithfulness measures whether the generated answer is fully supported by the retrieved context; groundedness measures whether every claim can be traced to a source.
PremBridge
Pega's secure hybrid gateway that routes LLM API calls through on-premises infrastructure, keeping sensitive data inside the enterprise perimeter while still using cloud-hosted models.
Knowledge Buddy
Pega's enterprise RAG product — indexes internal documents and knowledge bases, then surfaces grounded answers via a chat-style interface embedded in Pega applications.
Blueprint
Pega's AI-powered case design tool that translates natural language intent into a structured case type, data model, and process skeleton, dramatically accelerating low-code development.
Autopilot
Pega's agentic development feature that autonomously generates and modifies application rules in App Studio based on conversational instructions, enabling continuous AI-driven refinement.
Center of Excellence CoE
A cross-functional team (enterprise architects, data scientists, risk/compliance, product owners) that defines AI standards, governs model adoption, and enables GenAI scaling across the organisation.
AgentX
Pega's multi-agent orchestration framework — routes tasks to specialised sub-agents, manages tool invocations via MCP, and enforces HITL guardrails inside Pega workflows.
